CentOS 安装 Composer 并配置环境

2019-07-23 2 min read # composer # php包管理工具 # centos

😳PHP CLI

首先检测 PHP CLI 版本，输入命令

> php -v
PHP 5.2.3 (cli) ...

需要更换 PHP CLI 版本，使用 BT 面板中 7.2 的版本，设置一个别名

> vi ~/.bashrc

输入 i 进入编辑模式，新增一行

alias php='/www/server/php/72/bin/php'

输入 :wq 保存并退出

再次执行 php -v 查看版本，输出

PHP 7.2.4 (cli) (built: Nov 12 2018 10:47:44) ( NTS )
Copyright (c) 1997-2018 The PHP Group
Zend Engine v3.2.0, Copyright (c) 1998-2018 Zend Technologies

看到已经是 7.2.4 的版本

Composer

使用 PHP 下载安装文件

> php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"

验证文件是否正确

> php -r "if (hash_file('sha384', 'composer-setup.php') === '48e3236262b34d30969dca3c37281b3b4bbe3221bda826ac6a9a62d6444cdb0dcd0615698a5cbe587c3f0fe57a54d8f5') { echo 'Installer verified'; } else { echo 'Installer corrupt'; unlink('composer-setup.php'); } echo PHP_EOL;"

输出 Installer verified 则为正确，执行安装文件

> php composer-setup.php

接着删除安装文件

> php -r "unlink('composer-setup.php');"

移动到全局

> mv composer.phar /usr/local/bin/composer

中国镜像

> composer config -g repo.packagist composer https://packagist.phpcomposer.com
# 或使用阿里云镜像
> composer config -g repo.packagist composer https://mirrors.aliyun.com/composer/

Composer 命令

安装/更新项目依赖

> cd '你的项目地址'
> composer install
# 或
> composer update

如果报错

> Unsupported declare 'strict_types' ...

PHP版本错误，原因是因为直接使用 composer 命令应该调用到了老版本的 PHP CLI，添加一个别名即可

> vi ~/.bashrc

输入 i 进入编辑模式，新增一行

> alias composer='php /usr/local/bin/composer'

输入 :wq 保存并退出

重新登录 SSH 即可生效 .bashrc 文件
